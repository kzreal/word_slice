# Word 文档切片工具 - 项目概述

## 项目简介

Word 文档切片工具是一个基于 Python Flask 的 Web 应用，能够将 Word 文档（.docx）按目录结构自动切片为多个编号的 Markdown 文件。工具支持表格、图片的完整保留，并集成了 LLM（大语言模型）API 进行智能图片识别。

## 核心功能

### 1. 📄 文档切片
- **零级模式**：整个文档转成一个 Markdown 文件
- **按级别切片**：支持 1-3 级标题的章节切片
- **全部模式**：按所有标题层级切片
- **编号系统**：每行内容都有独立的行号标记（`<!-- 行号 -->`）

### 2. 🖼️ LLM 图片识别
- **自动识别**：检测文档中的图片并调用 LLM API 进行内容识别
- **格式统一**：识别结果以 `<!-- 行号 -->[图片: 描述]` 格式输出
- **批量处理**：优化性能，支持大批量图片处理
- **降级处理**：API 不可用时自动使用占位符，保证文档完整性

### 3. 📊 表格处理
- **完整保留**：表格结构完整转换为 Markdown
- **图片合并**：表格同一行的多张图片合并为一行输出
- **行号管理**：表格行和图片行都保持正确的行号连续性

### 4. 🔧 技术特性
- **Web 界面**：友好的上传和下载界面
- **大文件支持**：支持最大 1GB 的 Word 文档
- **环境变量配置**：灵活的 API 配置管理
- **详细日志**：完整的处理过程日志记录

## 技术架构

### 后端技术
- **Python Flask**：Web 框架
- **python-docx**：Word 文档解析
- **requests**：HTTP API 调用
- **python-dotenv**：环境变量管理

### 前端技术
- **HTML/CSS/JavaScript**：原生前端实现
- **FileReader API**：文件上传处理
- **Fetch API**：异步请求处理

### LLM 集成
- 支持任何兼容 OpenAI 格式的 API
- 自动 Base64 编码图片数据
- 智能重试机制和错误处理

## 环境配置

### 1. 安装依赖
```bash
pip install -r requirements.txt
```

### 2. 配置环境变量
```bash
# 复制模板文件
cp .env.example .env

# 编辑 .env 文件，填入 API 信息
LLM_API_ENDPOINT=https://your-api-endpoint.com/api/chat/completions
LLM_API_KEY=your-api-key
LLM_MODEL=qwen-vl  # 或其他支持图片的模型
LLM_TIMEOUT=30
LLM_MAX_RETRIES=3
```

## 使用方法

### Web 界面使用
1. 启动服务：`python3 tender_slicer_web.py`
2. 浏览器访问：http://localhost:8000
3. 上传 Word 文档
4. 选择切片级别（0/1/2/3/all）
5. 下载生成的 ZIP 文件

### 命令行测试
```bash
# 测试 LLM 配置
python3 test_llm_fixed.py
```
